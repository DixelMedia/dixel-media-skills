# Software Engineering

Esta categoría abarca buenas prácticas, arquitectura, patrones de refactorización y mentalidad pragmática al programar para mantener código mantenible y escalable.

## Contenido

- **clean-architecture**: Principios para independizar la interfaz, la base de datos y la configuración.
- **clean-code**: Buenas prácticas de código limpio y semántico.
- **ddia-systems**: Diseñar aplicaciones de datos intensivos (Designing Data-Intensive Applications).
- **domain-driven-design**: Reflejar el dominio de negocio real en la estructura del código.
- **high-perf-browser**: Técnicas y conceptos para exprimir el rendimiento de la web.
- **pragmatic-programmer**: Mentalidad, atajos y filosofía de un buen desarrollador (The Pragmatic Programmer).
- **react-best-practices**: Patrones, uso recomendado de hooks y escalabilidad en aplicaciones React.
- **refactoring-patterns**: Patrones de código para refactorizar deuda técnica.
- **release-it**: Ingeniería de software orientada a la vida en producción, despliegues y estabilidad.
- **software-design-philosophy**: Filosofía aplicada al diseño efectivo del software.
- **system-design**: Diseño y arquitectura general de sistemas escalables.
